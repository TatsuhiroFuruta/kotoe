# Kotoe デザインブリーフ集（Claude Design 貼り付け用）

各画面を **1枚ずつ** Claude Design に依頼するための文章。下の「貼り付け用」ブロックをそのままコピーして使う。

## 使い方のコツ
1. **最初に「0. 共通」を貼る** → 全体の雰囲気とグローバルナビが揃い、以降の画面が一貫する。
2. **そのあと画面を1枚ずつ**貼る（まとめて貼らない）。
3. 可能なら**画面遷移図のスクショ**を最初に添付すると、全体像が伝わって精度が上がる。
4. 生成後は「もっと余白を広く」「画像を大きく」などチャットで微調整。**色や雰囲気はここで決め込みすぎず、Design 上で反復**する。
5. 雰囲気の指定（下記の色・トーン）は好みで自由に変えてOK。

---

## 0. 共通（最初に貼る：トーン＋全体レイアウト）

> 貼り付け用：
> ```
> 「Kotoe（言絵）」というWebサービスのUIを作ります。画像を言葉で描写し、その言葉からAIが画像を再現して、再現度を競うゲームです。画像が主役なので、クリーンで余白を広めにとり、画像を引き立てる落ち着いた背景にしてください。アクセントカラーは1色（例：青緑）に絞り、全体はカード基調、角丸は控えめ。モバイルファーストのレスポンシブで。
> 共通のグローバルナビゲーション（ヘッダー）を作ってください：左にロゴ「Kotoe」、中央に「探す」「ランキング」、右に目立つ「お題を投稿」ボタンとユーザーアバター。フッターは簡素で。
> まずはこのヘッダー＋フッターと、全体のデザイントーンが分かるサンプルを見せてください。
> ```

## 1. トップ `/`

> 表示の要点：アプリ説明、遊び方、新着・人気のお題。
> 貼り付け用：
> ```
> Kotoeのトップページを作ってください。上から順に：(1)ヒーロー＝キャッチコピー1行＋サブ説明＋「お題に挑戦」「お題を投稿」の2つのCTAボタン。(2)「遊び方」3ステップ＝①画像を見る→②言葉で描写する→③AIが再現する、をアイコン付きで横並び。(3)「新着のお題」と「人気のお題」を、画像サムネイルのカードグリッドで。各カードは元画像・タイトル・挑戦数・いいね合計を表示。
> ```

## 2. お題一覧・検索 `/posts`

> 表示の要点：検索、並び替え、カードグリッド、ページネーション、空状態。
> 貼り付け用：
> ```
> お題の一覧ページを作ってください。上部に検索バーと「新着順／人気順」の並び替えトグル。本体は、お題カードのグリッド（各カード＝元画像サムネイル・タイトル・投稿者・挑戦数・いいね合計）。下部にページネーション。検索結果が0件のときの空状態（イラストと「条件を変えて探してみましょう」）も別バリエーションで見せてください。
> ```

## 3. お題詳細 `/posts/[id]`（最重要）

> 表示の要点：元画像＋ベスト再現（上位3件）＋描写入力（保存／生成の2ボタン）＋挑戦一覧。生成中・削除確認の状態も。
> 貼り付け用：
> ```
> お題の詳細ページを作ってください。上部：元画像を大きく表示＋タイトル＋投稿者＋「お気に入り」ボタン。その下に「このお題のベスト再現」セクション＝再現度（いいね数）が高い上位3件を表彰台風に強調表示（1位を中央に大きく、生成画像・挑戦者名・いいね数）。さらに下に「描写を書く」エリア＝複数行テキスト入力（プレースホルダーで書き方のヒント）と、「保存」「画像を生成」の2つのボタン（生成ボタンを主役として目立たせる）。いちばん下に「みんなの挑戦」一覧＝他ユーザーの挑戦カードのグリッド（生成画像・描写文の抜粋・いいね数・いいねボタン）、「再現度順（いいね順）／新着順」で並び替え可能。
> 追加で、(A)生成中の状態（ボタンがローディングになり「生成中…」のプレースホルダーカードが出る）、(B)自分の挑戦カードに表示される削除メニューと削除確認ダイアログ、も別バリエーションで見せてください。
> ```

## 4. 挑戦詳細・比較ビュー `/attempts/[id]`

> 表示の要点：元画像と再現画像の比較が主役。描写文、いいね、通報、共有。
> 貼り付け用：
> ```
> 挑戦の詳細（比較ビュー）ページを作ってください。主役は「元画像」と「AI再現画像」を左右に並べた比較表示（左右にスライドして重ねて見比べられるスライダー型も提案してほしい）。その下に、その挑戦者が書いた描写文を全文表示。サイドまたは下部に、いいねボタン＋いいね数、（あれば）類似度スコアのバッジ、共有ボタン、通報メニュー。上部に「このお題に戻る」導線。
> ```

## 5. お題投稿 `/posts/new`

> 表示の要点：画像アップロード、タイトル、投稿。アップロード中・エラー状態も。
> 貼り付け用：
> ```
> お題を投稿するページを作ってください。中央にカード型のフォーム：(1)画像アップロード欄＝ドラッグ＆ドロップ対応、選択後はプレビュー表示。(2)タイトル入力。(3)「投稿する」ボタン。アップロード中（プログレス）と、画像未選択などのバリデーションエラー時の表示も別バリエーションで。
> ```

## 6. ランキング `/rankings`

> 表示の要点：上位の強調＋リスト。ユーザー順位／挑戦順位の切替。
> 貼り付け用：
> ```
> ランキングページを作ってください。上部に「ユーザー順位／挑戦順位」の切替タブ。1〜3位は表彰台風に大きめのカードで強調（アバターまたは再現画像・名前・獲得いいね数）。4位以降は順位・サムネイル・名前/タイトル・いいね数のリスト行。下部にページネーション。
> ```

## 7. マイページ `/mypage`

> 表示の要点：プロフィール＋統計、タブ（投稿／挑戦／下書き／お気に入り）。
> 貼り付け用：
> ```
> マイページを作ってください。上部にプロフィールヘッダー（アバター・名前・統計＝投稿したお題数／挑戦数／獲得いいね合計）。その下にタブ「投稿したお題」「自分の挑戦」「下書き」「お気に入り」。「下書き」タブは、保存しただけでまだ生成していないプロンプト（お題サムネイル＋描写文＋「生成する」「編集」「削除」ボタン）のリスト。各タブは中身が空のときの空状態も見せてください。
> ```

## 8. ログイン／新規登録 `/login` `/signup`

> 表示の要点：シンプルな中央フォーム。
> 貼り付け用：
> ```
> ログインページと新規登録ページを作ってください。中央に最小限のカード型フォーム＝メールアドレスとパスワード、主ボタン、下部に「アカウントをお持ちでない方はこちら／既にお持ちの方はこちら」の切替リンク。将来用に「Googleでログイン」ボタンの場所も用意（任意）。エラー時のメッセージ表示も。
> ```
